## Phases

### Phase 18: GraphQL-Only Mode

**Goal**: Add `NAUTOBOT_MCP_GRAPHQL_ONLY` env var that restricts the MCP server to exposing only `graphql_query` and `graphql_introspect`.

**Plans**: 5 plans

Plans:

- [x] 18-01: Add GRAPHQL_ONLY_MODE constant + _ALLOWED_GQL_ONLY_TOOLS tuple to commands.py
- [x] 18-02: Implement two-layer enforcement in session_tools.py and middleware.py
- [x] 18-03: Create unit tests for GQLONLY-01 through GQLONLY-05
- [x] 18-04: Add UAT tests T-45, T-46, T-47 with auto-detection
- [x] 18-05: Document NAUTOBOT_MCP_ENABLE_ALL in CLAUDE.md and SKILL.md

**Details:**

Two-layer enforcement architecture:

1. **Manifest filter** (`session_tools.py`): `GQLOnlyTransform.list_tools()` returns only `graphql_query` and `graphql_introspect` when `GRAPHQL_ONLY_MODE=True`. Applied via FastMCP `Transform` at `create_app()` STEP 3b.

2. **Call blocker** (`middleware.py`): `ScopeGuardMiddleware.on_call_tool` raises `ToolNotFoundError` for any non-GraphQL tool call when `GRAPHQL_ONLY_MODE=True`.

**Key decisions:**
- `GRAPHQL_ONLY_MODE = os.environ.get("NAUTOBOT_MCP_ENABLE_ALL", "false").lower() != "true"` — GQL-only is the default
- Two-layer enforcement ensures defense in depth: manifest filter prevents tools from appearing; call blocker prevents direct access attempts
- UAT split into two scripts: `run_mcp_uat_all_tools.py` (45 tests) and `run_mcp_uat_gql_only.py` (2 tests)
