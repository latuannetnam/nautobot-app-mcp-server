# Requirements: nautobot-app-mcp-server

**Defined:** 2026-04-05
**Core Value:** AI agents can query Nautobot network inventory data via MCP tools with full Nautobot permission enforcement, zero extra network hops, and progressive tool discovery.

---

## v1.2.0 Requirements

Requirements for v1.2.0 milestone — separate-process MCP server refactor (Option A → Option B). All mechanical changes; existing capabilities (10 core tools, auth, pagination, MCPToolRegistry) transfer unchanged.

### Setup

- [ ] **P0-01**: Add `uvicorn >= 0.35.0` as explicit dependency in `pyproject.toml` (already transitive via FastMCP, needs explicit for `uvicorn.run()` API access)
- [ ] **P0-02**: Update `development/docker-compose.yml` to pass `NAUTOBOT_DB_*` env vars to MCP server container/service
- [ ] **P0-03**: Document `--workers 1` requirement in `docs/admin/upgrade.md` — multi-worker loses in-memory sessions (v2: Redis backend)

### Infrastructure

- [ ] **P1-01**: `start_mcp_server.py` production management command: `nautobot.setup()` → register tools → `mcp.run(transport="http")`; blocks forever; systemd-managed
- [ ] **P1-02**: `start_mcp_dev_server.py` dev management command: `create_app()` factory + `uvicorn.run(reload=True)` with file-watch restart
- [ ] **P1-03**: `create_app()` validates DB connectivity before FastMCP starts (query `extras_jobresult` to confirm DB is reachable)
- [ ] **P1-04**: Read `NAUTOBOT_CONFIG` and `PLUGINS_CONFIG` from environment variables

### Tool Registration

- [ ] **P2-01**: `@register_tool` decorator: dual registration — in-memory `dict` + FastMCP `.tool()` wiring; auto-generates JSON Schema from function signature via `inspect.signature`
- [ ] **P2-02**: `register_all_tools_with_mcp()` populates FastMCP from `MCPToolRegistry`; called at server startup
- [ ] **P2-03**: Nautobot plugin `ready()` generates `tool_registry.json` (replaces `post_migrate`; plugin runs in its own process)
- [ ] **P2-04**: All 10 core read tools refactored: `async def` + `sync_to_async(thread_sensitive=True)(get_sync_fn())`
- [ ] **P2-05**: No Django model imports at module level — lazy import inside tool functions only
- [ ] **P2-06**: Unit tests for `@register_tool` decorator and `register_all_tools_with_mcp()`

### Session State

- [ ] **P3-01**: Session state in `ctx.request_context.session` (FastMCP native dict, no monkey-patching)
- [ ] **P3-02**: `MCPSessionState` keyed by FastMCP `session_id` in `StreamableHTTPSessionManager.sessions`
- [ ] **P3-03**: `@scope_guard("dcim")` decorator replaces `mcp._list_tools_mcp` override — scope checked before tool executes
- [ ] **P3-04**: `mcp_enable_tools`, `mcp_disable_tools`, `mcp_list_tools` implemented in Option B pattern

### Auth

- [ ] **P4-01**: `get_user_from_request()` reads token from FastMCP request headers (`Authorization: Token <hex>`)
- [ ] **P4-02**: Token cached in `ctx.request_context.session["cached_user"]` (no monkey-patch; cache survives batched requests)
- [ ] **P4-03**: Nautobot `.restrict(user, "view")` on all querysets (preserved unchanged)
- [ ] **P4-04**: Document `proxy_set_header Authorization` for nginx production deployments

### Bridge Cleanup

- [ ] **P5-01**: Delete `nautobot_app_mcp_server/mcp/view.py` — entire WSGI→ASGI bridge removed
- [ ] **P5-02**: Delete `nautobot_app_mcp_server/mcp/server.py` — daemon thread, lazy factory, `_mcp_app` singleton gone
- [ ] **P5-03**: Remove MCP endpoint entry from `nautobot_app_mcp_server/urls.py` — endpoint is now on port 8005
- [ ] **P5-04**: Remove `mcp._list_tools_mcp` override from `MCPToolRegistry` initialization
- [ ] **P5-05**: Old endpoint (`/plugins/nautobot-app-mcp-server/mcp/`) returns HTTP 404
- [ ] **P5-06**: Update `SKILL.md` with new standalone endpoint URL (`localhost:8005/mcp/`)

### UAT & Validation

- [ ] **P6-01**: UAT scripts updated to hit new port (default 8005) — fix endpoint URL, fix auth tokens (40-char hex, no `nbapikey_` prefix), use `allow_redirects=False`
- [ ] **P6-02**: Token auth UAT passes: valid token → data returned, invalid token → empty result + warning logged
- [ ] **P6-03**: Session UAT passes: `mcp_enable_tools` → `mcp_list_tools` → tool list reflects enabled scope
- [ ] **P6-04**: All unit tests pass: `poetry run nautobot-server test nautobot_app_mcp_server`
- [ ] **P6-05**: UAT smoke test `docker exec ... python /source/scripts/run_mcp_uat.py` exits with code 0

## Out of Scope

Explicitly excluded. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Horizontal multi-worker scaling | Requires Redis session backend; `--workers 1` is sufficient for v1.2.0 |
| Real-time SSE streaming | Cursor pagination handles most cases; SSE for v2+ only |
| DRF integration | Nautobot has no DRF; standalone FastMCP removes the need for Django bridging entirely |
| `redis` session backend | Over-engineering for v1.2.0; in-memory sessions sufficient |
| Write tools | Permission surface widens significantly; deferred to v2 |
| Metaclass-based tool registry | `register_mcp_tool()` explicit API is sufficient; metaclass overhead not justified |

## v2.0 Requirements

Requirements for v2.0 milestone — GraphQL MCP tool via graphene-django.

### GraphQL Core

- [ ] **GQL-01**: User can execute arbitrary GraphQL queries via `graphql_query` MCP tool
- [ ] **GQL-02**: `graphql_query` uses `sync_to_async(thread_sensitive=True)` at the outer boundary
- [ ] **GQL-03**: `graphql_query` reuses `nautobot.core.graphql.execute_query()` — no parallel schema built
- [ ] **GQL-04**: `graphql_query` accepts `query: str` and `variables: dict | None` parameters
- [ ] **GQL-05**: `graphql_query` returns a dict with `data` and `errors` keys
- [ ] **GQL-06**: `graphql_query` passes `output_schema=None` to FastMCP tool decorator
- [ ] **GQL-07**: `graphql_query` auth token resolves via `get_user_from_request()` from FastMCP request context

### Schema Introspection

- [ ] **GQL-08**: User can introspect GraphQL schema via `graphql_introspect` MCP tool
- [ ] **GQL-09**: `graphql_introspect` returns GraphQL SDL string describing available Nautobot types

### Security & Hardening

- [ ] **GQL-10**: Query depth limit enforced (max_depth ≤ 8) to prevent deeply nested DoS
- [ ] **GQL-11**: Query complexity limit enforced (max_complexity ≤ 1000) to prevent expensive DoS
- [ ] **GQL-12**: GraphQL syntax errors returned as structured `errors` array, not HTTP 500s
- [ ] **GQL-13**: Permission enforcement verified — `AnonymousUser` gets empty results, authenticated user gets filtered results

### Testing

- [ ] **GQL-14**: Unit tests verify auth propagates correctly to GraphQL execution context
- [ ] **GQL-15**: Unit tests verify valid GraphQL query returns structured `{data, errors}` dict
- [ ] **GQL-16**: Unit tests verify invalid query returns errors dict
- [ ] **GQL-17**: Unit tests verify variables injection works correctly
- [ ] **GQL-18**: UAT smoke test P-09 in `scripts/test_mcp_simple.py` — `graphql_query` against core Nautobot model
- [ ] **GQL-19**: UAT full suite T-37+ in `scripts/run_mcp_uat.py` — 4+ GraphQL tests covering auth, introspection, errors

### Documentation

- [ ] **GQL-20**: `graphql_query` and `graphql_introspect` documented in SKILL.md with example queries

### Deferred to Future

| Feature | Reason |
|---------|--------|
| Custom GraphQL schema | Reuse `nautobot.core.graphql.schema` — parallel schema misses Nautobot's `extend_schema_type` dynamic features |
| GraphQL as URL endpoint | MCP tool IS the interface; `/graphql/` returns 404 by design |
| Write mutations | Focus is read-only GraphQL queries; mutations deferred |
| DataLoader / N+1 prevention | `OptimizedNautobotObjectType` handles common cases; complex multi-hop queries can be addressed in future |
| Multi-query execution | Single operation per request; multi-op support deferred |
| Query result caching | TTL cache deferred to future iteration |
| Dry-run / validation mode | `validate_only` flag deferred |
