```python
async def on_call_tool(
    self,
    context: MiddlewareContext[CallToolRequestParams],
    call_next,
) -> Any:
    params = context.message  # CallToolRequestParams — has .name (NOT context.method)
    registry = MCPToolRegistry.get_instance()
    tool = registry._tools.get(params.name)

    # GQL-only mode: block all non-GraphQL tools.
    # This is a security backstop — even if client bypasses manifest, call is blocked.
    if GRAPHQL_ONLY_MODE:
        if params.name not in _ALLOWED_GQL_ONLY_TOOLS:
            raise ToolNotFoundError(
                f"Tool '{params.name}' is not available in GraphQL-only mode. "
                f"Only graphql_query and graphql_introspect are available."
            )
        # Fall through to normal GraphQL handling (allow the call)

    # Core tools: always pass through (no scope check needed)
    if tool is None or tool.tier == "core":
        return await call_next(context)

    # [rest of existing scope checking logic unchanged — lines 62-82 in original]
```

The GQL-only block MUST come before the `if tool is None or tool.tier == "core"` check, so that even GraphQL tools with tier="core" are properly handled (but they will match the allowlist anyway). Non-GraphQL core tools (like mcp_enable_tools) will be blocked because they are not in _ALLOWED_GQL_ONLY_TOOLS.

Error message MUST be: "Tool '{params.name}' is not available in GraphQL-only mode. Only graphql_query and graphql_introspect are available."
