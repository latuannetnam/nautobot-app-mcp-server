## Env-var inheritance verification

`NAUTOBOT_DB_*` vars are passed via the `&nautobot-base` anchor which defines:

```yaml
x-nautobot-base: &nautobot-base
  image: "nautobot-app-mcp-server/nautobot:${NAUTOBOT_VER}-py${PYTHON_VER}"
  env_file:
    - "development.env"    # contains NAUTOBOT_DB_NAME, NAUTOBOT_DB_USER, NAUTOBOT_DB_HOST
    - "creds.env"          # contains NAUTOBOT_DB_PASSWORD
  tty: true
```

`development.env` sets: `NAUTOBOT_DB_NAME`, `NAUTOBOT_DB_USER`, `NAUTOBOT_DB_HOST`
`creds.env` sets: `NAUTOBOT_DB_PASSWORD`

All four required vars are covered. No duplication needed — the `mcp` service inherits `env_file` from the anchor.
