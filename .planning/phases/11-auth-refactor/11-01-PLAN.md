# Plan 11-01: Refactor `get_user_from_request()` to async + FastMCP state cache

**Phase:** 11-auth-refactor · Wave 1
**Requirements:** P4-01, P4-02
**Files touched:** `nautobot_app_mcp_server/mcp/auth.py`, `nautobot_app_mcp_server/mcp/tests/test_auth.py`

---

## Context

`get_user_from_request()` is currently a sync `def` using `_cached_user` attribute on `ctx.request_context`. Phase 10 established `ctx.get_state()`/`ctx.set_state()` as the canonical FastMCP session-state API (ServerSession has no dict interface). Phase 11 adopts that pattern.

Token source (`ctx.request_context.request.headers.get("Authorization", "")`) is already correct — no change needed there. The refactor is two-fold: make the function `async def`, and replace attribute-based caching with FastMCP state API.

---

## <threat_model>

| Threat ID | Description | Mitigation |
|-----------|-------------|------------|
| T-11-01 | Token cache bypass — cache miss not checked properly | Explicit `if cached_user_id is not None` guard; cache miss falls through to DB |
| T-11-02 | Session key collision — wrong session gets cached user | FastMCP auto-prefixes keys with `session_id` via `_make_state_key()` |
| T-11-03 | Non-serializable value in cache — user object stored | Always store `str(user.pk)`, never the user object; `serializable=True` (default) |
| T-11-04 | Stale cache — user deleted/deactivated but still returned | On cache hit, re-fetch user by ID from DB (`User.objects.get(pk=id)`) to validate |

</threat_model>

## Task 11-01-01: Refactor `auth.py` — make async, use FastMCP state API

**Steps:**

1. Add `_CACHED_USER_KEY = "mcp:cached_user"` constant at module level.
2. Change `def get_user_from_request(ctx)` → `async def get_user_from_request(ctx)`.
3. In the cache-hit path: `cached_user_id = await ctx.get_state(_CACHED_USER_KEY)`.
   - If non-None, fetch user by ID: `User.objects.get(pk=cached_user_id)`. Catch `User.DoesNotExist` and fall through to re-authenticate.
4. In the cache-miss path: after successful DB token lookup, store `await ctx.set_state(_CACHED_USER_KEY, str(user.pk))`.
5. Delete all `_cached_user` attribute access (`getattr`, `setattr`, direct assignment).
6. Update docstring to reflect async nature and FastMCP state cache.

**Verify:**
```bash
grep -n "^async def get_user_from_request" nautobot_app_mcp_server/mcp/auth.py
# → 1 match (function is async)
grep -n "mcp:cached_user" nautobot_app_mcp_server/mcp/auth.py
# → 2 matches (get + set)
grep -n "_cached_user" nautobot_app_mcp_server/mcp/auth.py
# → 0 matches (attribute removed)
grep -n "await ctx.get_state\|await ctx.set_state" nautobot_app_mcp_server/mcp/auth.py
# → 2 matches
grep -n "str(user.pk)" nautobot_app_mcp_server/mcp/auth.py
# → 1 match (stores string, not object)
```

## Task 11-01-02: Update `test_auth.py` for async API + new cache mechanism

**Steps:**

1. In `_make_mock_ctx()`, add `get_state`/`set_state` async mock methods backed by a shared `_store` dict (same pattern as Phase 10 `test_session_tools.py`):
   ```python
   _store: dict[str, str | None] = {}

   async def _mock_get_state(key: str):
       return _store.get(key)

   async def _mock_set_state(key: str, value: str):
       _store[key] = value

   mock_ctx.get_state = _mock_get_state     # type: ignore[method-assign]
   mock_ctx.set_state = _mock_set_state     # type: ignore[method-assign]
   ```
2. Rewrite `test_cached_user_returned_on_second_call`:
   - Set up cache via `await ctx.set_state("mcp:cached_user", str(user_obj.pk))`
   - First call returns cached user (re-fetched by ID)
   - No token DB lookup occurs on second call
3. Rewrite `test_cache_stores_user_after_db_lookup`:
   - Verify `await ctx.set_state("mcp:cached_user", str(user_obj.pk))` was called after DB lookup
4. Rewrite `test_cache_miss_falls_through_to_db`:
   - Verify `await ctx.get_state("mcp:cached_user")` returns `None` on first call
5. Add `test_cache_hit_re_validates_user`: verify that on cache hit, `User.objects.get(pk=id)` is called (DB query), not token lookup.
6. Add `test_cache_stores_user_id_not_object`: verify `set_state` is called with `str(user.pk)`, never the user object.
7. Convert all existing sync `get_user_from_request()` calls to use `async_to_sync()` wrapper (Django `TestCase` sync methods):
   ```python
   from asgiref.sync import async_to_sync
   def test_...(self):
       async def _run():
           return await get_user_from_request(ctx)
       result = async_to_sync(_run)()
   ```
   Applies to: `test_missing_authorization_header_returns_anonymous`, `test_missing_authorization_header_logs_warning`, `test_invalid_token_format_returns_anonymous`, `test_non_nbapikey_token_returns_anonymous`, `test_valid_token_returns_user`, `test_valid_token_wrong_key_returns_anonymous`, `test_empty_token_returns_anonymous`.

**Verify:**
```bash
grep -n "async_to_sync\|get_state\|set_state\|_CACHED_USER_KEY" \
  nautobot_app_mcp_server/mcp/tests/test_auth.py
# → all four appear
grep -n "_cached_user" nautobot_app_mcp_server/mcp/tests/test_auth.py
# → 0 matches (attribute removed from tests too)
```
