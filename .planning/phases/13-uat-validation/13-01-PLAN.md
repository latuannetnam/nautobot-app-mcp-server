# Plan 13-01: Docker Compose MCP Server Service

## Task: Wire the MCP server as a separate Docker Compose service

### Context

Phase 8 created the management commands (`start_mcp_server.py`, `start_mcp_dev_server.py`) but never wired the MCP server into the Docker Compose dev stack. The `mcp` service in `docker-compose.dev.yml` has `entrypoint: "tail -f /dev/null"` — a placeholder that does nothing.

This plan creates a proper `docker-compose.mcp.yml` override that defines a real `mcp-server` service, removes the placeholder `mcp` stub, and adds the new file to `tasks.py`'s `compose_files` list.

---

### Step A — Create `development/docker-compose.mcp.yml`

<read_first>

- `development/docker-compose.base.yml` lines 9–14 — `x-nautobot-base` anchor (image, env_file, tty)
- `development/docker-compose.dev.yml` lines 17–41 — real service examples to follow
- `development/docker-compose.dev.yml` lines 52–57 — `mcp` placeholder to replace
- `development/development.env` — all `NAUTOBOT_DB_*` vars already present

</read_first>

<action>

Create `development/docker-compose.mcp.yml` with this content:

```yaml
---
# docker-compose.mcp.yml — MCP server as a standalone service
# Included by tasks.py compose_files (invoke start) alongside dev/postgres/redis overrides.
x-nautobot-build: &nautobot-build
  build:
    args:
      NAUTOBOT_VER: "${NAUTOBOT_VER}"
      PYTHON_VER: "${PYTHON_VER}"
    context: "../"
    dockerfile: "development/Dockerfile"
x-nautobot-base: &nautobot-base
  image: "nautobot-app-mcp-server/nautobot:${NAUTOBOT_VER}-py${PYTHON_VER}"
  env_file:
    - "development.env"
    - "creds.env"
  tty: true

services:
  mcp-server:
    <<: *nautobot-base
    command: "nautobot-server start_mcp_dev_server"
    ports:
      - "8005:8005"
    depends_on:
      db:
        condition: service_healthy
    volumes:
      - "./nautobot_config.py:/opt/nautobot/nautobot_config.py"
      - "../:/source"
```

**Key design decisions:**
- Service name: `mcp-server` (distinct from the old placeholder `mcp`)
- `command: nautobot-server start_mcp_dev_server` — blocks forever, no `tail -f /dev/null`
- `ports: "8005:8005"` — host→container port mapping (already in dev compose; keep for clarity)
- `depends_on: db` with `service_healthy` — ensures DB is ready before MCP server starts
- Inherits `x-nautobot-base` anchor for image + env_file (same as nautobot/worker/beat)
- Volumes: `nautobot_config.py` (required for `nautobot.setup()`) + source code (`:/source`)

</action>
