# Plan 13-02: Update UAT Script to Port 8005

## Task: Change `run_mcp_uat.py` endpoint default to port 8005

### Context

After Phase 12, the MCP server runs as a **standalone FastMCP process** on port 8005. The old embedded endpoint at `http://localhost:8080/plugins/nautobot-app-mcp-server/mcp/` no longer exists (Phase 12 deleted `urls.py`). `run_mcp_uat.py` must point to the new endpoint.

---

### <read_first>

- `scripts/run_mcp_uat.py` lines 44–45 — current `DEV_URL` and `MCP_ENDPOINT`
- `scripts/run_mcp_uat.py` lines 46–49 — token fallback chain

</read_first>

### <action>

Change `scripts/run_mcp_uat.py` line 45:

**Before:**
```python
MCP_ENDPOINT = f"{DEV_URL}/plugins/nautobot-app-mcp-server/mcp/"
```

**After:**
```python
MCP_ENDPOINT = f"{DEV_URL}/mcp/"
```

**Rationale:**
- `DEV_URL` defaults to `http://localhost:8080` (Nautobot). The MCP server is on a different port (8005).
- Replace the entire `DEV_URL + embedded path` with the new standalone endpoint.
- Since `DEV_URL` is Nautobot's URL (port 8080) and the MCP server is on port 8005, remove the `DEV_URL` dependency for this value entirely.

**Alternative (cleaner — choose one):**

Option A — hardcode the new endpoint:
```python
MCP_ENDPOINT = "http://localhost:8005/mcp/"
```

Option B — keep env var but use MCP-specific var:
```python
MCP_DEV_URL = os.environ.get("MCP_DEV_URL", "http://localhost:8005")
MCP_ENDPOINT = f"{MCP_DEV_URL}/mcp/"
```

**Decision:** Use **Option B** — keep `MCP_DEV_URL` env var support so users can override to a different host/port if needed (e.g., remote dev box, different port).

```python
MCP_DEV_URL = os.environ.get("MCP_DEV_URL", "http://localhost:8005")
MCP_ENDPOINT = f"{MCP_DEV_URL}/mcp/"
```

Add `MCP_DEV_URL` to the `<environment_variables>` section comment in the script header (update the comment on lines 16–17):
```python
# Environment variables (from nautobot_import.env):
#     MCP_DEV_URL      MCP server URL (default: http://localhost:8005)
#     MCP_DEV_TOKEN     Dev auth token (default: from development/creds.env)
```

</action>
